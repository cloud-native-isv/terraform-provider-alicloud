# Feature Detail: Tablestore VCU Instance Support

**Feature ID**: 007  
**Name**: Tablestore VCU Instance Support  
**Description**: Support configuration of Tablestore instances using the VCU sizing model and elastic limits.  
**Status**: Draft  
**Created**: 2026-01-06  
**Last Updated**: 2026-01-06

## Overview

Tablestore now supports VCU (Virtual Compute Unit) pricing model offering serverless-like elasticity. The Terraform provider needs to support creating instances with this specification and configuring the elastic VCU upper limit.

## Latest Review

Initial design and specification phase.

## Key Changes

1. Add "VCU" to `instance_specification` validation (already present).
2. Ensure `elastic_vcu_upper_limit` is correctly handled in Create and Update operations.
3. Verify API calls use the correct parameters for VCU instances.
4. Support independent lifecycle for VCU parameters.

## Implementation Notes

- Uses `CreateInstance` or `CreateVCUInstance` from API.
- `elastic_vcu_upper_limit` is a float.
- Depends on `cws-lib-go` tablestore API support which seems present.

## Future Evolution Suggestions

- Support additional VCU-specific monitoring or auto-scaling configurations if API allows.
